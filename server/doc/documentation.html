Path Internal path Method Description
/ /getData GET bring number of products and orders, insert admin to the system.
/register1 POST Checks if you have entered a valid ID, passwords and email and keeps them in variables.
/register2 POST Checks whether you have entered city, street ,first name and last name, and insert new user to the users table in mysql.
/newcart POST Finds your latest cart, checks if it's closed -insert a new cart for the user and keep the id, and if it's open does nothing. else if there is no cart at all- insert a new one.
/login POST Checks if the ID is valid, then if the password is correct, keeps a user data and brings the user's shopping cart, login to the system.
/logout DELETE logout from the system. (destroy the session)
/shop /:cartid GET Brings all the products and the shopping cart of the user.
/addtocart POST Put in the cart the specific product selected with the quantity that the user put in.
/addfromcart POST Increases the quantity of the specific product +1
/search POST search in the shop - if there is something in the input search he filters the products, and if not he returns them all
/deletefromcart DELETE Deleting the specific product quantity from the cart -1.
/resetcart/:cartid DELETE Reset the customer cart from all the products.
/orders /userdata GET Brings details about the connected user (city and street).
/neworder POST Produces a new order, closes a current cart.
/admin /items GET Brings to the admin all the products.
/additem POST Add a product to the product list.
/:id PUT Update existing product information.
/types GET Brings all product categories for new product.
/getdata GET Brings specific product details for update.
/:cartid POST Create and open a receipt file.
/static/:cartid GET Download the receipt file.
